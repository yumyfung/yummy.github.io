工作流

工作流是区别于基础工具的，基础工具是不需要依赖任何工作记录来操作的，是独立的，随时可以使用，但是工作流是一个集成功能块的执行过程，它把本来需要多个步骤完成的过程构建成一个一步到位的自动化过程，工作流是依赖项目的，对一个项目进行高效处理。

目前工作流包含有：

  • 项目样式图片自动化一条龙服务(压缩->合并图片->base64->时间戳->...->上传文件到服务器)
  • 移动页面手机测试环境搭建（上传html并打开测试页面，使用的是团队自己的测试服务器，需要配置）
  • 生成提单文件（ars文件夹下）
  • 扩展创建活动模板（模板位于template文件夹下，是一个团队长期沉淀出来的，可以用来扩展到Smartgulp中，可以更加方便地创建项目）

在使用工作流命令前请先确保你的项目架构与Smartgulp相同，或者修改Smartgulp的配置路径使之符合你当前的文件架构。

工作流命令

提交样式到233服务器

gulp upload_233 [ -u ] [ -r ] [ -f XXX,XXXX,... ]	//可选参数-u（只更新有改动的文件），-r（同步资源，结合-f来用），-f（指定文件上传，后面的文件可以拖动到窗口获得文件路径，多个文件用逗号隔开）

提交项目到手机测试服务器

gulp upload_mobile -d 100 [ -u ] [ -r ] [ -f XXX,XXXX,... ] //同上

生成提单文件

gulp ars [ -u ]	//可选参数-u（只更新有改动的文件）

生成提单文件

gulp ars [ -u ]	//可选参数-u（只更新有改动的文件）

创建活动模板

gulp template_event -d 活动html文件夹名称,活动样式文件夹名称 [ -a 工作记录名 ]	//创建普通PC活动模板，可选参数-a，表示是否创建完成后是否顺便把它添加到工作记录中去，参数值为工作记录名
gulp template_mobile -d 活动html文件夹名称,活动样式文件夹名称 [ -a 工作记录名 ] //创建普通H5活动模板，可选参数-a同上
gulp template_slide -d 活动html文件夹名称,活动样式文件夹名称 [ -a 工作记录名 ]	//创建普通分屏滑动H5活动模板，可选参数-a同上
